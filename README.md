# Test_task
Задача: 
1. Завести проект на github или gitlab (на ваше усмотрение) 
2. Взять датасет MNIST из библиотеки torchvision 
3. Разбить датасет на OOD часть (примеры вне распределения), которая будет 
состоять из классов “0”, “1”, “2” и “3”, и на ID часть (примеры из распределения), 
для классов “4”-”9”. При этом обе части OOD и ID разбиваются на обучение и 
тест в соотношении 6 к 1 в соответствии со встроенным разбиением MNIST в 
библиотеки torchvision.  
4. Взять модель ResNet-18 из библиотеки torchvision предобученную на 
ImageNet, дообучить на ID части MNIST полученного в прошлом пункте (для 
этого надо заменить первый и последний слои, “заморозить” все внутренние 
слои и дообучить только первый и последний слой) 
5. Измерить точность (accuracy) полученной модели на тутовой выборки ID части 
(оно должно быть не ниже 85%) 
6. Далее реализовать Energy-based OOD детектор (работающий как бинарный 
классификатор, объект относится к ID данным или к OOD данным) из статьи “ 
Energy-based Out-of-distribution Detection” 
(https://proceedings.neurips.cc/paper/2020/file/f5496252609c43eb8a3d147ab9b9c00
6-Paper.pdf). PS порог отказа подбираете на основании обучающих частей ID и 
OOD выборок 
7. Оценить точность (accuracy) бинарной классификации на подвыборке 
состоящих из тестовых частей OOD и ID примеров исходной в соотношении 1 к 
1 из не менее чем 1000 изображений в сумме. (точность не ниже 70%)  
8. Реализовать FGSM атаку из статьи “Explaining and Harnessing Adversarial 
Examples” (https://arxiv.org/pdf/1412.6572) 
9. Применитть FGSM атаку к OOD части подвыборке из пункта 7. с целью их 
модификации для атаки на OOD детектор (для этого надо использовать 
градиенты детектора при прохождении через него этих изображений), чтобы 
выдать их за ID примеры. Значение гиперпараметра ε взять как 8/255 при 
диапазоне значений пикселей от 0 до 1 с шагом изменения интенсивности 1/255 
(это вариант изображений по умолчанию представлен в библиотеки torchvision, 
если будете использовать нормализацию, то значение ε необходимо 
скорректировать с учетом изменения диапазона значений для пикселей 
изображений) 
10. Повторно оценить точность (accuracy) бинарной классификации на подвыборке 
из пункта 7, где OOD часть была подвержена атаке в соответствии с пунктом 9. 
(деградация точности относительно значения полученного при выполнении 
пункта 7 должна составить не менее 10%)   
11. Пункты с расчетом метрик (5, 7 и 10) должны быть оформлены при сдаче в виде 
jupeter notebook с фиксированным random seed, который может задействовать 
код из этого же файла или из python файлов вашего репозитория
